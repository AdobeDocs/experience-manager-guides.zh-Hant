# 編輯器簡介 {#id181OD0QE05Z}

Experience Manager Guides隨附易於使用的編輯器，用於建立和編輯結構化檔案。 編輯器會隱藏DITA結構的所有複雜程度，不讓編寫器看到。 編輯器會提供使用者通常需要在檔案中工作的DITA元素清單。

## 新編輯器

>[!NOTE]
>
> Experience Manager Guides as a Cloud Service自2026.05.0版開始提供新編輯器。 請聯絡您的客戶成功團隊以啟用此功能。

新編輯器提供大幅演化的撰寫體驗，並著重改善效能、可用性和UI/UX。 專為DITA map和DITA主題而設計，可讓您有效率地撰寫大型、複雜和高度結構化的內容，同時保持完全向後相容且不影響非DITA內容。

### 主要功能

- 大型檔案的效能改善：透過支援復原/重做和髒汙標籤，增強大型DITA檔案的回應能力，確保可靠的變更追蹤而不會降低效能。
- 現代化表格製作：透過拖放重新排序、內容工具列和大量列或欄插入功能，提供更直覺式的表格編輯體驗，以更快且更精確的表格管理。
- 增強作者模式：提供在作者模式中直接可設定的XML註釋和屬性可見性，減少在使用結構化內容時切換到Source檢視的需求。
- 進階編輯模式與工具：針對主題引入並排式製作和Source模式，以及快速插入功能表，只需最少的上下文切換即可實現更快的結構化製作。
- UI和UX細分：透過深色主題和更加豐富的預設作者和預覽模式CSS，改善可讀性和視覺舒適度，提供更現代的編輯介面。
- 集中式編輯器設定：提供統一的設定面板以個人化編寫體驗，包括控制標籤、屬性、XML註解、不間斷空格和編輯器中的快速插入選單。

### 新編輯器的限制

新編輯器目前不支援下列功能。 這些功能預計將在即將發行的版本中推出。

- 在啟用追蹤變更的情況下製作（任何現有的追蹤變更都會保留，並視為一般內容）
- 新引文（內容中的任何現有引文都會保留）
- 「作者」檢視中的影像大小調整器
- 使用資料來源（例如JIRA、SQL資料庫、PIM或PLM）的資料

>[!NOTE]
>
>「新編輯器」不支援DITA map的「版面」檢視。


## 舊編輯器

Experience Manager Guides中的舊編輯器為結構化內容提供簡單而可靠的編寫體驗，抽象DITA的複雜性，同時確保遵循支援的標準，例如DITA 1.2、1.3、輕量型DITA和專門的DITA。 它提供使用通用區塊和內嵌元素的基本工具，因此非常適合在結構化架構中建立一致且受控制的內容。
