# 2026.05.0版中的已修正問題

本文介紹2026.05.0版Adobe Experience Manager Guides as a Cloud Service中不同區域修正的錯誤。

如需新功能和增強功能的詳細資訊，請檢視[2026.05.0版的新增功能](whats-new-2026-05-0.md)。

瞭解2026.05.0版[&#128279;](upgrade-instructions-2026-05-0.md)的升級指示。

## 編寫

- 使用&#x200B;**選取檔案**&#x200B;對話方塊在編輯器中選取影像時，只會顯示點陣格式（例如JPG、PNG和GIF）。 向量檔案（例如.ai和.eps）不會顯示且無法選取。 (GUIDES-45110)
- 升級時，`tagsView`設定預設為關閉，即使其`ui_config.json`中的預設值設定為`true`。 (GUIDES-44651)
- 在編輯器中，檔案參考會顯示為GUID，而非檔案路徑，無論是否有`xmleditor.uuid`設定。 (GUIDES-42438)
- 在DITA主題中套用具有類似鍵值的「主旨」配置時，它們會以幾乎相同的顏色突出顯示，因此很難區分它們並識別套用的配置。 (GUIDES-38472)
- 在作者檢視中編輯主題配置對應時，新增`hasInstance`元素會自動觸發檔案選擇對話方塊，要求作者插入指向AEM資產的不想要的`href`。 此外，無法為這類對應載入&#x200B;**內容屬性**&#x200B;面板，這會阻止作者更新作者檢視中的元素屬性，並要求他們使用Source檢視來更新屬性。 (GUIDES-38164)
- 編輯`.ditaval`檔案時，當您切換至「作者」檢視並返回「Source」檢視時，所有新增至「Source」檢視的XML註解都會被移除。 (GUIDES-33228)


## 資產管理

- 在名稱中包含空格的資料夾中建立主題時，會錯誤地建立重複的資料夾，其中空格會由連字型大小取代，且主題會儲存在那裡而不是原始資料夾。 (GUIDES-41938)
- 在第一次翻譯大型地圖時，會針對目標語言建立空的XML檔案，導致伺服器負載增加，效能降低。 (GUIDES-41613)
- 在DB型環境中，有效的內部DITA連結在&#x200B;**資產屬性**&#x200B;中會顯示為中斷連結，即使它們在編輯器和發佈的輸出中正常運作。 (GUIDES-35048)

## 發佈

- 當對資料夾設定檔中輸出預設集的變更套用至現有地圖時，已儲存的AEM Sites預設集的&#x200B;**發佈內容**&#x200B;會重設。 (GUIDES-38377)
- 當在地圖或書籤地圖的標題中使用`tm`元素時，原生PDF輸出的封面頁面上不會轉譯商標符號(®)。 (GUIDES-28832)
- 使用原生PDF範本發佈地圖時，**地圖示題**&#x200B;不包含地圖`title`內使用的子元素內容，且對應的內容篩選未套用至標題。(GUIDES-33730)
- AEM Sites輸出中的跨對應對等連結在指向使用`chunk="to-content"`的`topicref`時無法解析。 (GUIDES-37873)
- 在發佈期間，與DITAVAL型標幟相關的檔案會移至新的系統產生的資料夾，而非保留在輸出中預期的相對位置。 (GUIDES-37564)
- 使用多個具有衝突條件的DITAVAL檔案時，原生PDF輸出會失敗。 (GUIDES-37484)

## 基準線

- 使用自訂DITA-OT時，對映中的主題參照會錯誤地顯示為間接，即使它們被直接參照。 此問題已透過新的基準線體驗解決。 (GUIDES-19286)
- 具有`scope="peer"`的參考不正確地包含在基準內容中，導致發佈所需時間比預期長。 此問題已透過新的基準線體驗解決。 (GUIDES-30048)

## Platform

- 開啟大型主題或地圖時，製作執行個體會停止回應，在某些情況下需要重新啟動。 (GUIDES-43547)

## Editor 2.0可用的已解決問題

下列問題已修正，且使用編輯器2.0 （亦稱為新編輯器）時不再發生：

- 從表格中刪除兩個或多個欄時，表格結構會變得不一致或損毀。 (GUIDES-35438)
- 從包含合併儲存格的表格中刪除欄時，會新增一個空白欄。 (GUIDES-30147)
- 從階層連結功能表將新列插入現有表格時，表格結構會意外變更，導致遺失邊界和額外的欄。 (GUIDES-29194)
- 在「作者」檢視中，複製和貼上表格列會將內容置於表格之外，而非在表格中插入為新列。 (GUIDES-24372)
- 在製作模式中使用滑鼠拖曳選取的區段元素被複製並貼上時，會轉換為段落`(<p>)`元素，而非保留區段結構。 (GUIDES-30023)
- 編輯步驟或uicontrol等元素內反白顯示的文字時，選取的文字無法正確取代，而是附加或附加於開頭，導致驗證錯誤。 (GUIDES-24371)
- 使用相對值設定表格欄寬時，剩餘欄不會按比例調整，導致表格版面配置未對齊。 (GUIDES-26109)
- 在停用標籤的情況下貼上複製的主題標題時，第一次貼上會套用不正確的樣式，並將內容屬性中的型別指派為主題而不是標題。 (GUIDES-28838)
- 編輯大部分內容時，無意中捲動會導致編輯器檢視從使用中內容跳離。 (GUIDES-35436)
- 在元素上使用Backspace時，無論游標位置為何，編輯器都會捲動到主題的頂端。 (GUIDES-32520)
- 當使用左鍵或右鍵從內嵌標籤中移出時，游標在第一次嘗試時意外跳躍。 (GUIDES-26363)
- AEM Spellcheck僅適用於預設的en-US語言，不適用於其他地區。 (GUIDES-14731)
- 在編輯器中解鎖大型DITA主題時，同一主題會在重複標籤中重新開啟。 此外，會觸發標籤限制警告，其中顯示`NaN`而非實際標籤計數。 (GUIDES-34008)
- 針對唯讀或鎖定的主題，Acrolinx建議在編輯器中未正確反白顯示。 (GUIDES-29614)
- 在「作者」檢視中建立不含標題列的新`reltable`時，表格版面配置會在主題新增至第一個儲存格後變更，導致下一欄摺疊且難以放置相關主題。 (GUIDES-19555)
- 在「作者」模式中，將`xref`連結新增至小型表格儲存格時，該連結不會維持在儲存格內，而是會出現在相同列中相鄰儲存格之間。 （指南–5489）
- 從「作者」切換至「Source」檢視時，游標位置不會保留，且編輯器會捲動回頂端。 此外，在長主題中，當在Author和Source檢視之間切換時，游標位置會遺失並隨機跳至中間或頂端。 (GUIDES-18114， GUIDES-21164)
- 在`<li>`元素內按&#x200B;*Enter*&#x200B;會建立新的`<li>`，但導覽回到先前的`<li>`並按&#x200B;*Enter*&#x200B;會錯誤地將目前專案的內容轉換成`<p>`元素，破壞清單結構。 (GUIDES-27505)

## 已知問題

Adobe已找出下列2026.05.0版的已知問題：

- 當對應包含指向非DITA資源（例如`.html`或`.htm`）的外部`topicref`時，其預覽未顯示在Assets UI中。 (GUIDES-45409)
- 在Assets UI中，使用`conref`參照的內容無法針對DITA主題顯示，即使它在編輯器預覽中正確呈現。 (GUIDES-45505)<br>**因應措施**：對於這類內容，您可以使用編輯器的預覽。
- 啟用`xmleditor.uniquefilenames`屬性時，使用範本建立的新主題不包含主題標題。 (GUIDES-44737)
