# 已修正5.2.0版（2026年5月）中的問題

本文章說明5.2.0版Adobe Experience Manager Guides中不同區域所修正的錯誤。

如需新功能和增強功能的詳細資訊，請檢視[ 5.2.0版的新功能](whats-new-5-2-0.md)。

瞭解5.2.0版](upgrade-instructions-5-2-0.md)的[升級指示。


## 製作

- 如果將沒有屬性或值的空白`prop`元素新增到DITAVAL檔案，則無法新增其他`prop`元素。 (GUIDES-33597)
- 將Experience Manager Guides升級至2025.08.0版後，**Workspace設定**&#x200B;中不再顯示啟用或停用自訂&#x200B;**Acrolinx**&#x200B;索引標籤的選項。 (GUIDES-35261)
- 在瀏覽器重新整理時，在主題或地圖的資料夾層級設定檔中套用的自訂CSS將恢復為預覽模式的預設樣式。 (GUIDES-31098)
- 在DITA檔案編輯器的Source檢視中，**復原**&#x200B;和&#x200B;**取消復原**&#x200B;作業無法如預期運作。 (GUIDES-24914， GUIDES-25034)
- 使用`Xref`元素在主題中參照DITA map時，會顯示參照之對應的檔案名稱，而非對應的標題。 (GUIDES-21759)
- 透過Editor介面的右側面板新增多個Schematron檔案以進行驗證時，即使新增的檔案有效且沒有錯誤，也會顯示錯誤&#x200B;**Schematron檔案不存在或存在錯誤**。 (GUIDES-33731)
- 大型或複雜的MathML方程式無法在編輯器中顯示。 (GUIDES-29095)
- 當開啟多個DITA map或主題且其中一個主題關閉時，顯示所有開啟索引標籤的&#x200B;**>>**&#x200B;按鈕與索引標籤列上剩餘的開啟索引標籤重疊。 (GUIDES-31421)
- 啟用&#x200B;**核准工作流程**&#x200B;後，如果[左側]面板和[內容]屬性面板皆已開啟，編輯器工具列上就不會顯示&#x200B;**開始新版本**&#x200B;按鈕。 (GUIDES-29093)
- 當程式碼片段名稱超過程式碼片段面板的寬度時，它們會錯誤地將包裝在下一行，導致與相鄰的程式碼片段重疊，並影響可讀性。 (GUIDES-22685)
- 當您在DITA map中多次新增相同的參考時，**Map**&#x200B;檢視只會顯示最後一個事件的標題，而先前的檢視會顯示參考的UUID。 (GUIDES-9699)
- DITAVAL檔案保持可編輯狀態，即使其他使用者已鎖定它們，或伺服器已啟用&#x200B;**停用編輯，但不鎖定檔案**，且檔案是以唯讀模式開啟。 (GUIDES-27754)
- 正在從內部清理工作產生遺失節點的記錄，這些記錄不是必需的並且錯誤地標籤為錯誤，從而導致記錄檔案雜亂。 (GUIDES-31765)
- 當編輯結構描述(`*.sch`)檔案並使用尋找和取代功能時，尋找和取代面板會在底部部分顯示在熒幕外，阻止存取其輸入欄位和控制項。 (GUIDES-38412)
- 無法從&#x200B;**另存為新版本**&#x200B;對話方塊將多個&#x200B;**版本標籤**&#x200B;新增至主題。 (GUIDES-32716)
- 使用&#x200B;**選取檔案**&#x200B;對話方塊在編輯器中選取影像時，只會顯示點陣格式（例如JPG、PNG和GIF）。 向量檔案（例如.ai和.eps）不會顯示且無法選取。 (GUIDES-45110)
- 升級時，`tagsView`設定預設為關閉，即使其`ui_config.json`中的預設值設定為`true`。 (GUIDES-44651)
- 在編輯器中，檔案參考會顯示為GUID，而非檔案路徑，無論是否有`xmleditor.uuid`設定。 (GUIDES-42438)
- 在DITA主題中套用具有類似鍵值的「主旨」配置時，它們會以幾乎相同的顏色突出顯示，因此很難區分它們並識別套用的配置。 (GUIDES-38472)
- 在作者檢視中編輯主題配置對應時，新增`hasInstance`元素會自動觸發檔案選擇對話方塊，要求作者插入指向AEM資產的不想要的`href`。 此外，無法為這類對應載入&#x200B;**內容屬性**&#x200B;面板，這會阻止作者更新作者檢視中的元素屬性，並要求他們使用Source檢視來更新屬性。 (GUIDES-38164)
- 編輯`.ditaval`檔案時，當您切換至「作者」檢視並返回「Source」檢視時，所有新增至「Source」檢視的XML註解都會被移除。 (GUIDES-33228)
- 使用內容選單中的編輯MathML選項更新內嵌MathML方程式時，更新後的值要等到頁面重新整理後才會反映出來。 (GUIDES-38198)
- 當主題包含許多在「可重複使用」面板中新增的可重複使用元素（具有ID的元素）時，由於容器高度固定，有些元素可能無法存取。 (GUIDES-37220)
- 插入檔案的互動參照時，地圖和主題的圖示相同。 (GUIDES-36662)
- 編輯地圖時，`navtitle`中的特殊符號不會在Author檢視中顯示`topichead`。 (GUIDES-35435)

## 資產管理

- 當您透過Experience Manager Guides UI重新上傳已編輯的影像時，影像的原始轉譯會更新，但相關的DITA內容會繼續顯示影像的先前版本。 (GUIDES-33693)

- 當上傳檔案名稱包含無效字元的資產時，該資產無法上傳並顯示不正確的訊息&#x200B;**檔案已由其他使用者**&#x200B;鎖定，儘管該資產已解鎖。 (GUIDES-32680)

## Editor 2.0可用的已解決問題

下列問題已修正，且使用編輯器2.0 （亦稱為新編輯器）時不再發生：

- 從表格中刪除兩個或多個欄時，表格結構會變得不一致或損毀。 (GUIDES-35438)
- 從包含合併儲存格的表格中刪除欄時，會新增一個空白欄。 (GUIDES-30147)
- 從階層連結功能表將新列插入現有表格時，表格結構會意外變更，導致遺失邊界和額外的欄。 (GUIDES-29194)
- 在「作者」檢視中，複製和貼上表格列會將內容置於表格之外，而非在表格中插入為新列。 (GUIDES-24372)
- 在製作模式中使用滑鼠拖曳選取的區段元素被複製並貼上時，會轉換為段落`(<p>)`元素，而非保留區段結構。 (GUIDES-30023)

## 已知問題

### 製作

- 在DITAVAL檔案中上傳已標幟的影像時，當`Enable UUIDs`設定停用時，瀏覽器重新整理後影像會中斷。 (GUIDES-45853)
- 在編輯器中，啟用&#x200B;*核准工作流程*&#x200B;時，`.ditval`和`.md`檔案會變成不可編輯的檔案。 (GUIDES-42037)
- 如果主題位於書籤地圖標籤（前言、章節、部分或標題）或循環內容的一部分，在預覽模式中選取主題時，不會在地圖檢視中反白顯示主題。 (GUIDES-42416)
- 當檔案在「編輯器」和「搜尋」面板中開啟時，從「總管」面板中刪除它將會移除檔案並重新整理「總管」清單，但重新整理頁面會繼續在「搜尋」面板中顯示檔案。 (GUIDES-41935)

### 資產管理

- 在Assets UI中，當選取超過2個檔案或資料夾時，在第一次嘗試時不會啟用&#x200B;**移動**&#x200B;按鈕。 (GUIDES-42721) <br> **因應措施**：選取兩個以上的檔案或資料夾後，請等候幾秒鐘再選取目的地資料夾。

### 檢閱

- 更新作用中稽核任務時，如果已經是稽核一部分的某個主題被移除，然後在不按一下「更新」的情況下重新新增，則「稽核者」標籤中的稽核者資訊會遺失。 (GUIDES-38774)
- 從進行中的稽核任務中移除稽核中的主題時，其檔案狀態會繼續保持&#x200B;**在稽核中**，即使該主題不再是任何稽核任務的一部分。 (GUIDES-38709)<br>**因應措施**：從[屬性]頁面或[檔案屬性]面板，將主題的檔案狀態從[稽核中] **變更成適當的狀態**。

### 編輯器2.0

- 將相同主題中的內容複製貼上到編輯器中的無效位置時，會插入意外的外來標籤。 (GUIDES-45378)

- 包裹在`foreign`和`equation-block`標籤內的MathML方程式會引入不必要的空格並中斷編輯行為。 (GUIDES-46606)
- 拖放包含索引鍵參考的元素會將`keyref`轉換為絕對路徑。 (GUIDES-45701)
- 在對應編輯器中，中斷連結上的`Ctrl+click`會觸發應用程式錯誤。 (GUIDES-45544)
